32 **OpenESS** is a network-capable sound server libary mainly for embedded systems and other operatins systems.
33 OpenESS is free and open-source software, and is licensed under the terms of the GNU Lesser General Public License.

37 ## Getting Started
38 
39 ### Setting Up Development Environment
40 
41 #### PlatformIO
42 
43 OpenESS is made for use with [platformio](http://platformio.org/), an advanced ecosystem for microcontroller
44 development. To get started with coding esphomelib applications, you first have to
45 [install the atom-based platformio IDE](http://platformio.org/platformio-ide) or for advanced users,
46 [install the command line version of platformio](http://docs.platformio.org/en/latest/installation.html).
47 
48 Then create a new project for an [ESP32-based board](http://docs.platformio.org/en/latest/platforms/espressif32.html#boards)
49 (for example, `nodemcu-32s`). Then open up the newly created `platformio.ini` file and insert
50 
51 ```ini
52 ; ...
53 platform = espressif32
54 board = nodemcu-32s
55 framework = esp-idf
56 lib_deps = openess
57 ```
58 Finally, create a new source file in the `src/` folder (for example `main.c`) and start coding with openess.

60 ## Usage example
61 
62 _create the audio context_
63 ```cpp
64 #include "ess_context.h"
65 #include "ess_format.h"
66 
67 #include <stdio.h>
68 #include <stdlib.h>
69 #include <unistd.h>
70 
71 
72 void app_main() {
73  ess_context_t context;
74  ess_context_error_t error;
75 
76  error = ess_context_create (&context, ESS_BACKEND_NAME_I2S_ESP32, ESS_FORMAT_STEREO_44100_16);
77  if(error != ESS_CONTEXT_ERROR_OK) printf("error in creating the context\n");
78 
79  for(;;) { usleep(100000); }
80 }
81 
82 
83 ```
84 _For more examples and usage, please refer to the [Wiki][wiki]

86 ## Current Features (version 0.2-1)
87 
88 * Powerful core that allows for easy to port
89 * Automatic WiFi handling (reconnects, etc.)
90 * Powerful socket abscrations layer (SAL)
91 * Easy to use platform configuration
92 * Semaphore, task and ringbuffer handling on various platform
93 * generic backends: udp, uart and i2s
94
